学生向け — シミュレーターの使い方

このドキュメントは学生が本シミュレーターを使って学習するための簡易手順書です。実際の OS インストールは行わない疑似体験であり、操作は一本道で学習に集中できるよう設計されています。

1. クイックスタート
 - 最も簡単: `index.html` をブラウザで開く（ファイルをダブルクリックで可）
 - ローカルサーバ推奨（例: Python を使う）:

```powershell
python -m http.server 8000
# ブラウザで開く: http://localhost:8000/
```

2. 画面の流れ（学生が行う操作）
 - 「インストールメディアを挿入」→ 次へ
 - ブートメニューで "Install AlmaLinux" を選ぶ（画面の指示に従う）
 - 言語はデフォルトの日本語のままで進めて良い
 - インストール概要で項目（インストール先、ソフトウェア、ユーザー作成）を順に入力
 - ユーザー作成ではパスワードのバリデーションに従う（パスワードはローカルでのみ一時的に扱われ、記録はされません）
 - 「インストール開始」を押すと進捗画面に移り、フェイクログとプログレスバーが表示されます

3. 進捗の保存
 - インストール完了画面やデスクトップ画面にて「進捗を保存（JSON ダウンロード）」ボタンがある場合、これを押して `progress.json` をダウンロードしてください。
 - `progress.json` は授業中の提出や講師への報告に使用します。ファイルの中にパスワード等の機密情報は含まれません。

4. よくある質問
 - Q: パスワードは保存されますか？
   - A: いいえ。学習用のため、パスワードは保存しません。認証チェックは固定のテストアカウントやローカル検証用のダミーデータを使います。
 - Q: 間違えて閉じてしまったら？
   - A: 進行中のセッションは復元されません。もう一度最初から行ってください（授業では途中保存を促す運用を推奨します）。

5. トラブルシューティング
 - ページが白い/動かない: ブラウザのコンソール（F12）を確認してください。依存ファイルが同フォルダにあることを確認します。
 - ダウンロードボタンが動作しない: ブラウザのポップアップやダウンロード設定を確認してください。

6. 提出方法（授業）
 - ダウンロードした `progress.json` を講師の指示に従って提出してください（メール、学内 LMS、もしくは直接渡す）。

7. 補足
 - ブラウザ版は学習用の疑似体験です。実際のディスク操作や OS インストールは一切行われません。
