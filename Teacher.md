講師向け — シミュレーター運用ガイド

このドキュメントは講師が授業で本シミュレータを運用し、学生の進捗を収集・評価するための手順をまとめたものです。

1. 概要
 - 学生はブラウザ版（`index.html`）あるいは配布したデスクトップ版で演習を行います。
 - 学生は演習の最後に `progress.json` をダウンロードし、講師に提出します。

2. 教室準備（推奨環境）
 - 受講者用 PC に最新のブラウザ（Chrome/Edge/Firefox）を用意
 - インターネット接続は不要だが、ブラウザでローカルファイルを開けること
 - 必要に応じてローカルサーバ（Python の `http.server` 等）を用意すると静的ファイルの動作が安定します

3. 授業の流れ（例）
 - 講師が目的と学習目標を説明（約5分）
 - デモで操作手順を見せる（ブラウザで `index.html` を開いて実例）
 - 学生が個別にシミュレーションを実行（20〜40分）
 - 学生が `progress.json` をダウンロードして提出（LMS/メール/USB など）
 - 講師が `teacher.html`（本リポジトリ内）で複数の `progress.json` を集計し、CSV を出力

4. 進捗回収と集計（`teacher.html` の使用）
 - `teacher.html` をブラウザで開き、学生から回収した `progress.json` を複数選択してアップロードします。
 - アップロード後、「集計」ボタンで一覧を表示し、CSV 形式でダウンロード可能です。
 - この処理はブラウザ内で完結し、外部サーバにデータは送信されません。

5. 評価の例
 - 到達ステップ数（最後に到達したステップ）をもとに合否判定の一助にする
 - 特定ステップ（ユーザー作成やインストール開始）に到達しているかをチェック
 - コメント欄や手動での観察で操作の理解度を補完

6. 配布方法（授業で使う場合）
 - 推奨: 事前に ZIP にまとめて配布（`index.html`, `style.css`, `app.js`, assets）
 - 大人数の場合は教室内サーバ（ローカルの簡易 HTTP サーバ）で配信すると管理が楽です

7. よくあるトラブルへの対処
 - 学生が JSON をダウンロードできない: ブラウザのダウンロード権限やポップアップ設定を確認
 - 動作が遅い/固まる: 別のブラウザで試す、キャッシュをクリアする

8. セキュリティとプライバシー
 - `progress.json` は操作の到達履歴のみを含み、パスワード等の機密情報は含みません。
 - 集計はローカルで行い、外部送信しない運用を推奨します。

9. 拡張案
 - 講師向けの解説パネル（ヒント・模範解答）を追加して、困った学生にオンデマンドで表示
 - 学生の操作ログをリアルタイムで収集する簡易サーバ（ただしプライバシー配慮が必要）
